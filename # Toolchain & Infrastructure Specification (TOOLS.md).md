# Toolchain & Infrastructure Specification (TOOLS.md)
**Проект:** Интерактивная платформа «TELL ME PLEASE»

Данный документ описывает утвержденный стек инструментов, внешних API и библиотек. Любое отклонение от этого списка требует согласования с Functional Validator.

## 1. Frontend Tools (Клиентская часть)
* **Core Framework:** Next.js (App Router), React.
* **Styling & UI:** TailwindCSS.
* **Animations:** Framer Motion (для создания интерактивного и геймифицированного интерфейса для 5-9 классов).
* **Распознавание речи (STT):** `Web Speech API` (`window.SpeechRecognition`).
  * **Почему:** Бесплатный нативный браузерный API, который транскрибирует речь локально без нагрузки на наш сервер [1, 2]. Идеально справляется с акцентами при наличии стабильного интернета у пользователя. Поддерживает промежуточные результаты (interim results) для мгновенного отображения текста [1, 2].
* **Воспроизведение аудио (Playback):** `AudioWorklet` из Web Audio API. 
  * **Почему:** Необходим для создания кастомного джиттер-буфера (Jitter buffer), который позволяет бесшовно склеивать и воспроизводить поступающие по WebSocket аудио-чанки без пауз.

## 2. Backend Tools (Серверная часть)
* **Core Framework:** FastAPI (Python 3.10+).
  * **Почему:** Нативная поддержка асинхронности (`async/await`) и веб-сокетов, что критически важно для потоковой передачи аудио и текста с минимальной задержкой.
* **HTTP Клиент:** `httpx` или `aiohttp` для асинхронных неблокирующих запросов к LLM и TTS.

## 3. LLM Provider (Генерация текста)
* **Провайдер:** OpenCode Zen [3, 4].
* **Endpoint:** `https://opencode.ai/zen/v1/chat/completions` [3, 4].
* **Модель:** `deepseek-v4-flash-free` [4].
* **Особенности интеграции:** 
  * Запросы должны использовать стандартный формат OpenAI.
  * Бэкенд должен реализовывать **Key Rotation Manager** (переключение пула ключей из `.env` при получении HTTP 429/403) для обхода лимитов бесплатного тарифа.
  * Обязательно использование параметра `stream=True` для потоковой посимвольной выдачи текста.

## 4. TTS Engine (Синтез речи)
* **Движок:** `Kokoro-FastAPI` [5].
* **Реализация:** Локальный Docker-контейнер, эмулирующий OpenAI-совместимый API для модели Kokoro-82M [5].
* **Почему:** Kokoro-82M — открытая модель (Apache 2.0) весом менее 1 ГБ, генерирующая аудио коммерческого качества [6, 7]. На GPU выдает скорость до 100x от реального времени, на мощном CPU задержка составляет менее 1 секунды [5, 7]. Поддерживает авто-нарезку (auto-stitching) предложений [5].
* **Рекомендуемые голоса для AI-аватара:** `af_bella` (теплый и дружелюбный), `af_sky` (легкий и понятный) или `af_nicole` (четкий и профессиональный) [7]. Голоса жестко задаются в конфигурации бэкенда.

## 5. Дополнительные утилиты
* **VAD (Voice Activity Detection):** На клиенте опираемся на события окончания речи (`onresult`, `onend`) от Web Speech API [2]. Для защиты от "эха" (когда микрофон ловит голос из динамиков) обязательно реализуется программный `mute` микрофона на клиенте в моменты работы `AudioWorklet`.
