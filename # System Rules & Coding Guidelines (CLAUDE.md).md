# System Rules & Coding Guidelines (CLAUDE.md)
**Проект:** Интерактивная платформа «TELL ME PLEASE»

## 1. Роли и методология (Loop Engineering)
* **Твоя роль:** Lead AI Agent (Maker). Ты несешь полную ответственность за написание, тестирование и интеграцию кода.
* **Роль пользователя:** Functional Validator. Пользователь НЕ пишет код, он только проверяет работоспособность (UI/API) и дает обратную связь. Не проси пользователя исправлять код вручную.
* **Гигиена контекста (Ralph Loop):** Перед началом любой задачи обязательно читай `Plan.md` и `Status.md`. После успешного завершения задачи — атомарно обновляй `Status.md`, чтобы зафиксировать состояние системы для будущих сессий.
* **Лимиты:** При возникновении зацикленных ошибок или падении тестов 5 раз подряд, останови генерацию и запроси инструкции у Валидатора.

## 2. Архитектурные запреты (CRITICAL BANS)
* **ЗАПРЕЩЕНА ДИНАМИЧЕСКАЯ ГЕНЕРАЦИЯ ПРОМПТОВ В LLM:** Модель (`deepseek-v4-flash-free`) работает ИСКЛЮЧИТЕЛЬНО как чат-бот. Вся логика маршрутизации и выбор системного промпта (для 5-9 классов) должна быть жестко закодирована (hardcoded) в скриптах бэкенда на основе `branch_id`, получаемого от клиента.
* **ЗАПРЕЩЕН SERVER-SIDE STT:** Не пытайся интегрировать Whisper или другие тяжелые модели распознавания речи на бэкенд. Захват и транскрибация голоса выполняются ТОЛЬКО на клиенте с помощью бесплатного нативного `Web Speech API`.
* **ЗАПРЕЩЕНЫ УТЕЧКИ API-КЛЮЧЕЙ:** Никогда не храни и не передавай API-ключи от OpenCode Zen на фронтенд. Все запросы к LLM строго проксируются через FastAPI-бэкенд.
* **ЗАПРЕЩЕНЫ ПОСЛЕДОВАТЕЛЬНЫЕ ЗАДЕРЖКИ:** Не дожидайся завершения генерации всего ответа от LLM. Обязательно используй архитектуру Streaming Pipeline: Потоковый токен от LLM -> Буфер предложений (Sentence Buffer) -> TTS-генерация в Kokoro -> WebSocket -> Непрерывное воспроизведение на клиенте.

## 3. Требования к стеку технологий
* **Frontend:** Next.js (App Router), TailwindCSS, Framer Motion. Для бесшовного воспроизведения потоковых аудио-чанков без "заиканий" используй Vanilla JS `AudioWorklet`.
* **Backend:** Python 3.10+, FastAPI. Строго используй асинхронный код (`async/await`) для всех эндпоинтов и WebSocket-соединений. 
* **State Management (LLM):** Поддерживай "скользящее окно контекста", сохраняя только последние 10-12 реплик диалога, чтобы избежать ошибки переполнения в 200 000 токенов на бесплатных ключах.

## 4. Key Rotation Manager
* Бэкенд ОБЯЗАН включать в себя менеджер ротации ключей. При получении ошибки `429 Too Many Requests` или `403` от шлюза OpenCode Zen, скрипт должен "на лету" (без разрыва WebSocket-соединения с клиентом) переключиться на следующий API-ключ из заранее заданного пула и повторить запрос.

## 5. Предотвращение акустического эха
* На клиенте обязательно реализовать глушение (mute) микрофона или паузу записи в Web Speech API на время физического воспроизведения синтезированной речи из динамиков, чтобы избежать попадания аудио обратно в STT-движок.
