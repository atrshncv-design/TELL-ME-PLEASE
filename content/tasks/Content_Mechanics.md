# Content Mechanics — План механик игр

## Категории упражнений

| Категория | Раздел на фронте | Описание |
|-----------|------------------|----------|
| grammar | Грамматика | Окончания, V1/Vs, отрицание, вопросы, наречия |
| vocabulary | Словарный запас | Тематическая лексика (распорядок дня, животные) |
| listening | Аудирование | Role-play диалоги, прослушивание |
| speaking | Общение | Голосовой ввод, диалог с AI-аватаром |

## Механики по типам заданий

### 1. Drag-and-Drop (Сортировка окончаний, Наречия)
- **UI**: Три колонки с drop-зонами. Карточки с глаголами перетаскиваются мышкой или тач-жестом.
- **Framer Motion**: `layout` + `AnimatePresence` для плавного перемещения. `whileDrag={{ scale: 1.05 }}`.
- **Валидация**: Проверка после каждого размещения или по кнопке "Проверить".
- **Feedback**: Зелёная анимация при правильном ответе, красная при ошибке.

### 2. Quiz — выбор из вариантов (V1/Vs, Шутка, Отрицание, Do/Does)
- **UI**: Кнопки-варианты под предложением. Один правильный ответ.
- **Framer Motion**: `whileTap={{ scale: 0.95 }}` на кнопках, `motion.div` с `initial/animate` для появления вопроса.
- **Прогресс**: Полоска прогресса сверху (сколько вопросов пройдено).
- **Шутки**: Дополнительно — emoji-анимация при правильном ответе.

### 3. Fill-in (Рулетка, Наречия-место, Гарри Поттер)
- **UI**: Предложение с пропуском (кликабельное поле или dropdown).
- **Framer Motion**: Текст появляется по буквам при правильном ответе.
- **Валидация**: Проверка по кнопке или автоматически при заполнении.

### 4. Ladder (Лесенка Up and Down)
- **UI**: Вертикальная лесенка. Ученик наступает на ступеньку — она подсвечивается и предложение зачитывается.
- **Framer Motion**: `layout` анимация при переходе между ступеньками. Подсветка текущей ступеньки.
- **Направление**: Утро — снизу вверх (от короткого к длинному). Вечер — сверху вниз.

### 5. Build Sentence (Наречия: построй)
- **UI**: Банк слов (наречия + время + глагол). Ученик собирает предложение кликами.
- **Framer Motion**: Слова "прилипают" к строке предложения. Проверка по кнопке.

### 6. Role-Play (Интервью с Гарри Поттером)
- **UI**: Два колонки (Журналист / Гарри). Реплики появляются по очереди. Подсветка текущей реплики.
- **Framer Motion**: Появление реплик с `initial={{ x: -20, opacity: 0 }}`.
- **TTS**: Озвучка реплик (TTS для каждого персонажа — разные голоса).

### 7. Fill-in-and-Speak (Расскажи о себе)
- **UI**: Промпты-подсказки с пропусками. Ученик заполняет и проговаривает.
- **STT**: Web Speech API распознаёт речь и заполняет пропуски.
- **Feedback**: Оценка произношения через LLM.

### 8. Voice-Chat (Поговори со сверстником)
- **UI**: Тот же экран чата с AI-аватаром, но с фиксированными темами.
- **STT + LLM**: Голосовой ввод → текст → ответ AI → озвучка.
- **Темы**: Утро, Школа, Вечер, Выходные.

## Определения механик (если не указано)

| Тип | Механика | Компонент |
|-----|----------|-----------|
| drag-and-drop | Перетаскивание карточек в колонки | `useDragControls` + `motion.div` |
| quiz | Клик по варианту ответа | `motion.button` + `whileTap` |
| fill-in | Ввод текста или выбор из dropdown | `input` / `select` + валидация |
| ladder | Последовательный переход по ступенькам | `useInView` + `AnimatePresence` |
| build-sentence | Клик по словам из банка | `useReducer` для строки |
| role-play | Поочерёдное появление реплик | `setTimeout` + `AnimatePresence` |
| fill-in-and-speak | Ввод + распознавание речи | `useSpeechRecognition` |
| voice-chat | Полный цикл STT → LLM → TTS | `useWebSocket` + `useSpeechRecognition` |
